## 📊 数据流

### 收件箱数据流
```
用户访问收件箱
  ↓
加载测试数据（立即显示）
  ↓
并行加载真实数据（从 API）
  ↓
混合显示（按日期分组）
  ↓
用户操作（删除、标记等）
  ↓
同步到后端 API
  ↓
刷新列表
```

### 邮件详情数据流
```
用户点击邮件
  ↓
判断邮件 ID 类型
  ↓
如果是真实数据 ID → 从 API 加载
如果是测试数据 ID → 使用本地测试数据
  ↓
显示邮件详情
```

## 📝 使用说明

### 查看测试数据
- 默认显示 6 封测试邮件 + 15 封更早邮件
- 测试数据带蓝色"测试"标签
- 点击测试数据邮件显示预设的测试内容

### 查看真实数据
- 点击"生成测试数据"按钮生成真实数据
- 真实数据会混合显示到对应日期分组
- 点击真实数据邮件显示从 API 加载的内容

### 操作邮件
- **删除**: 支持删除测试数据和真实数据
- **标记已读**: 支持标记真实数据
- **星标**: 支持标记真实数据
- **转发**: 仅前端模拟（可扩展）
- **垃圾邮件**: 仅前端模拟（可扩展）

## 🔧 技术细节

### 数据类型定义
```typescript
interface Email {
  id: number | string;  // 支持数字和字符串 ID
  from: string;
  subject: string;
  date: string;
  size: string;
  hasAttachment?: boolean;
  isRead: boolean;
  isStarred: boolean;
  avatar?: string;
  avatarColor?: string;
  isTest?: boolean;  // 标记是否为测试数据
}
```

### 日期格式化
- 今天：显示时间（HH:mm）
- 7 天内：显示星期（周一、周二等）
- 更早：显示日期（月 日）

### 混合显示逻辑

// 今天的邮件
{mockEmails.slice(0, 1).map(renderEmailRow)}
